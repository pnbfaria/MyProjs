How it Works

Watson X embeds AI agents via a JavaScript widget that connects to IBM's cloud infrastructure. The widget is configured with a hostURL (cloud endpoint) and an agentId (unique agent identifier), establishing a direct communication channel between the browser and the AI agent backend.

Configuration Analysis
FUJITSU CONFIDENTIAL

Watson X Widget Configuration

The Watson X AI Agent is embedded using a JavaScript snippet containing two critical parameters:

JavaScript
window.watsonAssistantChatOptions = {
    integrationID: "abc123-integration-id",
    region: "eu-de",
    serviceInstanceID: "instance-uuid-here",

    // Key discovery parameters:
    hostURL: "https://eu-de.assistant.watson.cloud.ibm.com",
    agentId: "agent-uuid-identifier",
};

hostURL

The base URL of the Watson X instance running on IBM Cloud. Determines the regional endpoint (e.g., eu-de for Frankfurt).

agentId

The unique identifier of the specific AI agent. This binds the widget to a particular agent's configuration, skills, and knowledge base.

Integration Model
FUJITSU CONFIDENTIAL

Watson X Agent Integration

1. Widget Initialization

The JavaScript snippet loads and initializes the Watson X chat widget with the configured parameters.

2. Endpoint Resolution

The hostURL resolves to the correct IBM Cloud regional data center (e.g., Frankfurt for eu-de).

3. Agent Binding

The agentId binds the widget session to the specific AI agent with its skills, actions, and knowledge base.

4. Session Management

Watson X manages the conversation session, maintaining context and state across interactions.

Key Findings & Next Steps

Discovery Outcomes
FUJITSU CONFIDENTIAL

Key Findings

1
Endpoint Successfully Identified: The Watson X AI agent operates via a regional IBM Cloud endpoint. The hostURL determines the data center, while the agentId specifies the exact agent configuration.
2
JavaScript Widget Architecture: Watson X uses a client-side JavaScript widget pattern for embedding. Configuration is passed via the watsonAssistantChatOptions global object, making it straightforward to analyze and integrate.
3
EU Data Residency: The eu-de region code confirms the data is hosted in Frankfurt, Germany, aligning with European data protection requirements (GDPR compliance).
4
Integration Flexibility: The agent can be embedded in any web application through the JS snippet, providing a low-friction integration path for existing enterprise web platforms.
